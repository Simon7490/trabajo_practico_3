Teoría

Programación

“El término programación se define como un conjunto de instrucciones consecutivas y ordenadas que llevan a ejecutar una tarea específica. Dichas instrucciones se denominan 'código fuente', el cual es único para cada lenguaje y está diseñado para cumplir una función o propósito específico.” (Morales, 2014)

Lenguaje de Programación

Un lenguaje de programación es una “estructura que, con una cierta base sintáctica y semántica, imparte distintas instrucciones a un programa de computadora” (Perez y Merino, 2009).

Un lenguaje de programación está compuesto por:

  • **Símbolos**
  • **Reglas sintácticas y semánticas**
  • **Estructura**
  • **Significado**
  • **Elementos y expresiones**

IDE (Entorno de Desarrollo Integrado)

Los IDE (Integrated Development Environment) son entornos de desarrollo donde es posible escribir, compilar (o interpretar) y ejecutar código en un lenguaje de programación, todo en el mismo programa.

Estos entornos facilitan el desarrollo de aplicaciones al incorporar:

  • **Herramientas de Debug:** tanto textual (mediante una consola) como gráfica.
  • **Bibliotecas:** componentes ya desarrollados para ser utilizados en el programa.
  • **Interfaz gráfica:** que facilita y agiliza la escritura de los programas.
  • **Funcionalidades adicionales:** acceso y gestión de bases de datos, integración con herramientas de control de versiones, simulación de servidores, generación de documentación, gestión de interfaces de usuarios, modelado de artefactos, etc.

Java

**¿Qué es Java?** Java es una tecnología que incluye un lenguaje de programación y una plataforma de desarrollo.

**La plataforma Java** Por definición, una plataforma es el ambiente hardware o software sobre el cual se puede ejecutar un programa.

  • **Plataforma Windows:** Un PC con su sistema operativo Windows.
  • **Plataforma Linux:** Una notebook con una distribución Linux.
  • **Plataforma Android:** Un celular con Android.

La plataforma Java se diferencia porque es una plataforma software que se ejecuta sobre las prestaciones que brinda otra plataforma que gestiona el hardware (plataforma base).

La plataforma Java tiene dos componentes principales:

  • **Java Virtual Machine (JVM):** Es un intérprete que traduce el bytecode al código máquina de la plataforma base al momento de ejecutar la aplicación.
  • **Java Application Programming Interfaces (API):** Una colección de componentes software organizados en bibliotecas (packages) que proveen muchas capacidades útiles para el desarrollador.

**El lenguaje de programación Java** Java es un lenguaje de programación de alto nivel definido por sus características: sencillo, orientado a objetos, distribuido, interpretado, robusto, seguro, independiente de las arquitecturas, portable, eficaz, multitarea y dinámico.

Componentes del lenguaje de programación Java

En el lenguaje de programación Java, todo el código fuente se escribe en archivos de texto plano con la extensión .java. Estos archivos fuente son entonces compilados por una herramienta de la plataforma denominada compilador javac, guardando el resultado en archivos con el mismo nombre pero con extensión .class. Un archivo .class no contiene código nativo para el procesador de la plataforma base, sino que contiene un tipo de código denominado bytecode (lenguaje binario intermedio). Los bytecode se pueden considerar el lenguaje máquina de la JVM, debido a que la JVM es la encargada de traducir los bytecode en el código máquina del ordenador.

Para realizar esta operación, la plataforma cuenta con una herramienta denominada java launcher, la cual es la encargada de crear una instancia de la JVM y ejecutar la aplicación en esa instancia. El proceso de ejecución de una aplicación java se denomina pseudo interpretado, debido a que existe una primera etapa de compilación a código intermedio (bytecode) y luego una segunda etapa donde se interpreta el código intermedio mediante la JVM.

Esta forma de ejecución permite utilizar una misma aplicación (los bytecode) en diferentes sistemas operativos; siempre y cuando ese sistema operativo disponga de la JVM. Algunas JVM, tales como Java SE HotSpot (la versión creada por Oracle) incorporan tareas adicionales en tiempo de ejecución que aumentan el rendimiento de la aplicación. Entre estas tareas se incluyen encontrar cuellos de botella en el rendimiento y volver a recompilar (en código nativo) secciones de código frecuente.

Características de Java

  • Sencillo: La sintaxis de Java es similar a la de los lenguajes C y C++, pero evita características semánticas que los vuelven complejos, confusos y poco seguros.
    • En Java sólo existen tres tipos primitivos: los numéricos (enteros y reales), el tipo carácter y el tipo booleano. Todos los tipos numéricos están firmados.
    • En Java, los arreglos, las matrices y las cadenas de caracteres son objetos, lo que facilita su creación y manipulación.
    • En Java, el programador no tiene que preocuparse de la gestión de la memoria. Un sistema denominado "el recolector de basura" se encarga de asignar la memoria necesaria a la hora de crear objetos y de liberarla cuando estos ya no se referencian en el dominio del programa.
    • En Java, no existen preprocesadores ni archivos de encabezamiento. Las instrucciones “define” de C se sustituyen por constantes en Java y las instrucciones “typedef” de C se reemplazan por clases.
    • En C y C++, se definen estructuras y uniones para representar tipos de datos complejos, mientras que, en Java, se crean instancias de clases para representar tipos de datos complejos.
    • En C++, una clase puede heredar varias clases, lo que puede generar problemas de ambigüedad. Con el fin de evitar estos problemas, Java sólo autoriza la herencia simple pero aporta un mecanismo de simulación de herencia múltiple mediante la implementación de una o varias interfaces.
    • En Java no existe la famosa instrucción goto, simplemente porque aporta una complejidad a la lectura de los programas y porque a menudo se puede prescindir de esta instrucción escribiendo un código más limpio. En Java, se utilizan las instrucciones break y continue, que permiten salir de uno o varios niveles de anidamiento.
    • En Java, no es posible sobrecargar los operadores, para evitar problemas de incomprensión del programa. Se preferirá crear clases con métodos y variables de instancia.
    • En Java, no hay punteros sino referencias a objetos o celdas de un arreglo o matriz (referenciadas por su índice), para evitar errores comunes en C y C++.
  • Orientado a objetos: Salvo los tipos de datos primitivos, todo en Java es concebido como un objeto. Java se ha provisto de clases incorporadas que encapsulan los tipos primitivos (los wrappers). Esto facilita un mejor dominio de la complejidad, una reutilización más sencilla y una mayor facilidad de corrección y evolución. Java estándar está dotado de un conjunto de clases que permiten crear y manipular todo tipo de objetos (interfaz gráfica, acceso a la red, gestión de entradas/salidas, etc.).
    • Distribuido: Java implementa protocolos de red estándar para desarrollar aplicaciones cliente/servidor en arquitecturas distribuidas, con APIs específicas.
    • Interpretado: La ejecución se realiza mediante la JVM, lo que permite portabilidad sin necesidad de recompilar.
    • Robusto: Java es fuertemente tipado, verificando errores en compilación y ejecución, y gestiona referencias de objetos para evitar errores.
    • Seguro: El JRE se encarga de la seguridad, controlando el acceso a memoria y redes, sin acceso directo a punteros.
    • Independiente de Arquitecturas: El compilador genera bytecode independiente de arquitecturas, que la JVM traduce a código nativo.
    • Portable: Java es portable gracias a su interpretación, y los tipos de datos primitivos mantienen su tamaño en diferentes plataformas.
    • Eficaz: Aunque interpretado, Java utiliza JIT para compilar bytecode a código nativo, alcanzando rendimiento similar a C/C++.
    • Multitarea: Java permite ejecutar múltiples hilos simultáneamente, aumentando la velocidad de las aplicaciones.
    • Dinámico: No requiere editar vínculos, permitiendo modificar clases sin actualizar todo el programa.

    Plataforma Java:

    • JRE: Ejecuta aplicaciones Java, instalando la JVM y APIs necesarias, con herramientas de despliegue.
    • JDK: Incluye componentes del JRE y agrega el lenguaje Java y herramientas de desarrollo.
    • Ediciones: Java SE, JavaFX, Java EE y Java ME, cada una con enfoques específicos y diferentes servicios y herramientas.
